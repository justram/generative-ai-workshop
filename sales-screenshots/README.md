# 銷售截圖說明

這組截圖可以用來快速說明這個工作坊的價值：它不是單純介紹 AI 名詞，而是讓學員在自己的電腦上，透過實際互動理解生成式 AI 的工作方式、限制與應用場景。

## 01-curriculum-overview.png

工作坊課程總覽。  
用來說明這是一套有結構的實作型課程，內容涵蓋大型語言模型基礎、聊天機器人、提示工程、文件與圖片上傳、成本、工具呼叫，以及本地模型等主題。

## 02-local-gpt-models.png

本地版 GPT 模型選擇頁。  
用來說明這個版本可以讓學員使用自己的 ChatGPT 帳號登入，不需要講師提供共用 API Key，也不需要幫學員管理任何登入密鑰或後端服務。

## 03-chatbot-interface.png

聊天機器人介面如何運作。  
用來說明學員會看到聊天介面背後實際送給模型的內容，理解 system prompt、上下文、訊息格式，以及為什麼聊天機器人看起來像在「理解」，但本質上仍是文字輸入與 token 輸出。

## 04-document-upload.png

文件上傳與文件問答。  
用來說明學員會理解 PDF、簡報、試算表、文件被轉成文字後會發生什麼事，包括格式、圖表、表格、頁碼、公式等資訊可能如何遺失，進而理解「跟文件聊天」的實務限制。

## 05-hidden-costs.png

隱藏成本與 token 累積。  
用來說明工作坊會讓學員看到每一輪對話、系統提示、歷史訊息、附件與模型輸出如何累積成 token 與成本，避免把「問一句話」誤解成只有一句話的成本。

## 06-tool-invocation.png

工具呼叫與 AI 自動化風險。  
用來說明課程不只停留在聊天，而是會進一步介紹模型如何呼叫工具、串接外部程式，以及在 email assistant 這類自動化場景中可能出現的安全邊界與風險。

## 建議銷售說法

這個工作坊適合想要從「會用 ChatGPT」進一步理解「生成式 AI 系統如何運作」的團隊。學員不只會學提示技巧，也會理解模型限制、文件處理、圖片上傳、成本累積、工具呼叫與本地部署等實務議題。

這個版本的重點是：學員可以下載一個獨立 App，在自己的電腦上登入自己的 ChatGPT 帳號開始使用。講師不需要提供 API Key，也不需要維護共用後端或管理學員的登入秘密。
